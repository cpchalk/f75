# This file encodes the figures 4.1 to 4.32 contained in the technical report
# 'The infinite Fibonacci groups and relative asphericity'
# by M.Edjvet and A.Juhasz
# 4-sided regions are described by the 4 vertices, joining the 4 edges, which are 
# encountered in the clockwise direction. A vertex is described by its successive angles 
# read in a clockwise direction.
# The angles are represented by letters a,A,b,B,x,X,y,Y,z,Z,l,m. These correspond to 
# the letters of the technical report, a, a bar, b, b bar, x, x bar, y, y bar, z, z bar,
# lamda,mu. The first letter of a vertex label is the angle 
# contained in the corner of the region being described read in a clockwise direction.
# Successive vertices are separated by a comma and describe an edge joining the two vertices, 
# the start vertex and end vertex of the edge read in a clockwise direction from within the 
# region being described.
# If the described edge receives a charge n from the region, the  end vertex label is 
# appended with the digit n. 
# When vertices are defined, they are given a number M, allocated in ascending order. 
# This number starts from 1 in each configuration. 
# If a vertex is referred to again, then it is written in the form Mp where p is the letter 
# contained in the corner of the region being described.
# If not all angles of a vertex are known, then the description n. is used. This indicates >=n more
# edges (read in a clockwise direction). So this represents n+1 extra angles. 
# The description n= indicates that there are exactly n more edges.
# Two or more regions are separated by ';''s. Each configuration has the label 4.nr where 4.n precedes the
# group of configurations and r precedes the description of regions.
# Charges are normally specified by an integer representing the number of 1/30'ths of charge. If another
# fraction is involved then the format NDDD is used to represent N/DDD eg 1012 represents 1/12.
